Add a **real preview page** that (1) fetches the generated video, (2) validates its resolution ≥ 720p and fps ≥ 24, (3) shows the video with native controls, (4) displays the actual resolution/FPS, (5) offers a **Refine** button that emits a `refineVideo` event for the rest of the app, and (6) falls back to clear error messages when anything goes wrong.

## 1. Diagnosis (merged)

| Symptom | Root cause (from both proposals) |
|---------|-----------------------------------|
| **No preview** – UI only shows a `<div>` placeholder. | `video_preview.html` never contains a `<video>` element. |
| **No video source** – JS stub never calls the backend. | `video_preview.js` has no fetch logic. |
| **No quality guard** – Users can see low‑res clips. | No validation of width/height/fps. |
| **No refinement UI** – Users can’t trigger a “refine” workflow. | No button or event wiring. |
| **No feedback** – When something fails the page stays blank. | No loader / error messages. |

## 2. Solution Overview  

| Component | What we add |
|-----------|-------------|
| **HTML** (`public/video_preview.html`) | `<video>` with controls, a loader, a “Refine Video” button, and a small metadata bar (resolution / FPS). |
| **Constants** (`src/constants.js`) | Export `MIN_RESOLUTION` (720 × 480) and `MIN_FPS` (24). |
| **API helper** (`src/api.js`) | `getGeneratedVideo(videoId)` → `GET /api/videos/:id` returning `{url, width, height, fps}`. |
| **JS logic** (`src/video_preview.js`) | • Parse `?id=` from the URL.<br>• Call the API helper.<br>• Validate quality against the constants.<br>• Populate the `<video>` source, show the metadata, hide the loader.<br>• Emit `refineVideo` when the button is clicked.<br>• Graceful error handling. |

All four files are **self‑contained** and use ES‑modules (`type="module"`), so the existing build pipeline (`npm run dev` / `vite` or similar) picks them up automatically.

### 1. Acceptance Criteria  
1. **UI Rendering** – A `<video>` element with id `preview-video` is rendered in `video_preview.html` when a video file is generated.  
2. **Source Binding** – The `src` attribute of the `<video>` element is set to the generated video URL (`/generated/<id>.mp4`).  
3. **Playback Controls** – The video element exposes native controls (`controls` attribute) and can be played, paused, and seeked.  
4. **Quality Validation** – On load, the video’s resolution is ≥ 720p and frame rate ≥ 30fps.  
5. **Refinement Trigger** – A “Refine” button (`#refine-btn`) is enabled only after the video has successfully loaded and passes quality checks.  
6. **Error Handling** – If the video fails to load or does not meet quality standards, an error banner (`#error-banner`) is displayed and the “Refine” button is disabled.

### 2. Unit Tests (JavaScript – Jest + jsdom)

```js
// video_preview.test.js
import { render, screen, fireEvent } from '@testing-library/dom';
import { initVideoPreview } from '../src/video_preview.js';

describe('Video Preview Component', () => {
  beforeEach(() => {
    document.body.innerHTML = `
      <video id="preview-video" controls></video>
      <button id="refine-btn" disabled>Refine</button>
      <div id="error-banner" hidden></div>
    `;
  });

  test('renders video element with correct id', () => {
    const video = screen.getByTestId('preview-video');
    expect(video).toBeInTheDocument();
  });

  test('sets src to generated video URL', () => {
    initVideoPreview('/generated/123.mp4');
    const video = screen.getByTestId('preview-video');
    expect(video.src).toContain('/generated/123.mp4');
  });

  test('enables refine button after successful load', async () => {
    initVideoPreview('/generated/123.mp4');
    const video = screen.getByTestId('preview-video');
    const refineBtn = screen.getByTestId('refine-btn');

    // Simulate successful load event
    fireEvent.loadedmetadata(video);
    await Promise.resolve(); // wait for async handlers

    expect(refineBtn).not.toBeDisabled();
  });

  test('displays error banner if video fails to load', async () => {
    initVideoPreview('/generated/invalid.mp4');
    const video = screen.getByTestId('preview-video');
    const errorBanner = screen.getByTestId('error-banner');
    const refineBtn = screen.getByTestId('refine-btn');

    fireEvent.error(video);
    await Promise.resolve();

    expect(errorBanner).not.toBeHidden();
    expect(refineBtn).toBeDisabled();
  });

  test('validates resolution and frame rate', async () => {
    initVideoPreview('/generated/123.mp4');
    const video = screen.getByTestId('preview-video');

    // Mock video metadata
    Object.defineProperty(video, 'videoWidth', { value: 1280 });
    Object.defineProperty(video, 'videoHeight', { value: 720 });
    Object.defineProperty(video, 'videoFrameRate', { value: 30 });

    fireEvent.loadedmetadata(video);
    await Promise.resolve();

    const refineBtn = screen.getByTestId('refine-btn');
    expect(refineBtn).not.toBeDisabled();
  });
});
```
