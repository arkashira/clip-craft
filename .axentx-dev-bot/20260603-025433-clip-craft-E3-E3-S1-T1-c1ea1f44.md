## 1. Acceptance Criteria (measurable)

1. **Data Capture** – Every video creation event must record:
   - `video_id`
   - `user_id`
   - `creation_timestamp`
   - `duration_seconds`
   - `engagement_metrics` (likes, comments, shares, view_count)

2. **Accuracy** – The stored values must match the source data within a tolerance of ±0.01 seconds for duration and ±1 for integer engagement counts.

3. **Freshness** – Analytics data must be available for querying within 5 seconds of the event completion.

4. **Filtering & Sorting** – API endpoint `/analytics` must support:
   - `filter[user_id]`, `filter[date_from]`, `filter[date_to]`
   - `sort[by]` (`creation_timestamp`, `duration_seconds`, `engagement_metrics`)
   - `sort[order]` (`asc`, `desc`)

5. **Pagination** – Results must be paginated with `page` and `page_size` parameters, default `page_size=50`.

6. **Security** – Only authenticated users can query analytics, and users can only see data for videos they own or have been granted access to.
